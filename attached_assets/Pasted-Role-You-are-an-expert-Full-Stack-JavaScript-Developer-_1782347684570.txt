2. Key Generation Function:
Create a utility function to generate the required headers for every request:

JavaScript
const crypto = require('crypto');

function getUgHeaders() {
  // Generate a random 16-character hex string for the device ID
  const deviceId = crypto.randomBytes(8).toString('hex'); 
  
  // Format current UTC time exactly as "YYYY-MM-DD:H" (e.g., "2022-05-13:20")
  const now = new Date();
  const year = now.getUTCFullYear();
  const month = String(now.getUTCMonth() + 1).padStart(2, '0');
  const day = String(now.getUTCDate()).padStart(2, '0');
  const hour = now.getUTCHours();
  const formattedDate = `${year}-${month}-${day}:${hour}`;
  
  // The string to hash MUST be exactly: deviceId + formattedDate + "createLog()"
  const strToHash = `${deviceId}${formattedDate}createLog()`;
  const apiKey = crypto.createHash('md5').update(strToHash).digest('hex');

  return {
    "Accept-Charset": "utf-8",
    "Accept": "application/json",
    "User-Agent": "UGT_ANDROID/4.11.1 (Pixel; 8.1.0)",
    "X-UG-CLIENT-ID": deviceId,
    "X-UG-API-KEY": apiKey
  };
}
3. Express API Routes for UG:

GET /api/ug/search?title=...: Fetches /tab/search?title={title}&page=1&type[]=Chords.

GET /api/ug/tab/:id: Fetches /tab/info?tab_id={id}&tab_access_type=private. Parse the response and extract the raw chord/lyrics string to save to the database.

GET /api/ug/explore: Fetches /tab/explore?order=hitsdailygroup_desc&type[]=Chords to show today's most popular tabs.

GET /api/ug/random: Fetches /tab/random to pull a random song chart.

PHASE 3: REAL-TIME WEBSOCKETS (HOST & PARTICIPANT)
Integrate Socket.io into the Node server to handle real-time gig synchronization. Do NOT sync Metronome ticks (latency makes this unusable). Sync ONLY navigation and scroll events.

Rooms: When a user clicks "Become Host", the server generates a 4-digit HOST CODE and creates a Socket room.

Participant Join: A user clicks "Join Host", enters the 4-digit code, and joins the room.

Sync Events: * song_change: Host changes a song, server broadcasts the Song ID to participants.

scroll_sync: Host scrubs the scroll bar, server broadcasts { position: percentage, speed: active_speed }.

Participant Overrides: The React frontend for Participants must allow them to toggle "Override Host View". If toggled on, the Participant device ignores incoming zoom/column changes but continues to listen to song_change and scroll_sync so the song stays roughly in sync.
